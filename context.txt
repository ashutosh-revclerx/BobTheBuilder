### Architecture

#### Canvas & Grid
- react-grid-layout Responsive + WidthProvider, 12-column grid
- Components stored as flat array in Zustand with parentId pointers for nesting
- GridLayer.tsx maps Zustand components → RGL layout, scoped by parentId
- **Freestyle Engine**: `compactType={null}` and `preventCollision={true}` are active.
- **Constraints**: Every component has `minW` and `minH` enforced in the store and mapped to RGL.
- **Auto-expansion**: Canvas height calculates dynamically based on `lowestPoint * (rowHeight + margin)`.
- Drag handle tied to `.canvas-component-label` floating pill.
- onLayoutChange → updateLayouts() → Zustand.

#### Component shape in Zustand
{
  id: 'barchart-12345678',
  type: 'BarChart',
  label: 'New BarChart',
  parentId: undefined,       // undefined = root canvas
  parentTab: undefined,      // for TabbedContainer tabs
  layout: { x: 0, y: 0, w: 6, h: 6, minW: 4, minH: 6 },
  style: { ... },
  data: { ... }
}

#### Drag from Sidebar → Canvas (implemented)
- LeftPanel cards: draggable=true, onDragStart sets dataTransfer componentType
- Global draggingType in Zustand (setDraggingType)
- GridLayer: isDroppable={!!draggingType}, droppingItem, onDrop → addComponent
- Drop zone shows dashed blue outline when drag is active
- .react-grid-placeholder styled blue dashed

#### Sidebar click-to-add (fallback, still works)
- onClick on LeftPanel card → addComponent(type)
- If a container is selected → auto-assigns parentId

### Features Implemented (do not regress these)

1. TRUE DRAG FROM SIDEBAR TO CANVAS
   - Drag any component card from LeftPanel onto canvas
   - Drops at exact grid coordinates
   - New component auto-selected, right panel opens

2. RIGHT PANEL BEHAVIOR
   - Opens on page load showing first component
   - Stays open when canvas background is clicked
   - Shows last selected component config even when nothing active
   - Only closes via X button (closeRightPanel action)
   - Zustand fields: rightPanelOpen (bool), lastSelectedComponentId

3. TABLE ROW ADDITION — two surfaces
   - Inline "+ Add Row" button at table bottom (editor mode only)
   - Right panel Data tab Rows section with add/delete/edit per row
   - Cells are inline-editable via contentEditable in editor mode
   - All writes go to component data.rows in Zustand

4. COMPONENT VERTICAL STRETCH
   - Components sit inside `.component-inner-content` (Flexbox wrapper).
   - `.component-inner-content` has `overflow: visible` to avoid clipping resize handles.
   - Inner component (.stat-card, .table-component) has `height: 100%` and `overflow: hidden`.
   - Charts use `ResponsiveContainer height="100%"` and parent div with `flex: 1; min-height: 0`.

5. RESIZE FROM ALL FOUR CORNERS
   - `resizeHandles={['se','sw','ne','nw']}` explicitly set on `ResponsiveGridLayout`.
   - Custom handle dots styled in index.css (blue dots, white ring).
   - Handles use `z-index: 60` to stay above all other UI layers (loading overlaps, etc).
   - Conflicting legacy RGL CSS (bottom/right anchors) must stay removed.

6. COMPONENT LABEL + DELETE BUTTON
   - .canvas-component-wrapper has position:relative, overflow:visible
   - .canvas-component-label: position:absolute, top:-11px, left:10px
   - .remove-btn: position:absolute, top:-8px, right:-8px
   - NEVER overflow:hidden on wrapper — it clips the floating elements.

7. STABLE DRAG FEEDBACK
   - `.react-grid-item` transitions limited to `left, top, width, height`.
   - `.react-draggable-dragging` state has `transition: none !important`.

### Known Recurring Bugs (watch for these)

A. overflow:hidden on .canvas-component-wrapper clips labels and handles.
   Always keep overflow:visible on the wrapper and intermediate content div.

B. `.react-grid-item > * { height:100% }` wildcard catches floating labels.
   Avoid wildcards; target specific component classes.

C. z-index conflicts: Loading overlays (z-50) can block resize handles if handles are low.
   Keep handles at z-60.

D. RGL defaults to `['se']` handle only. Always explicitly set all 4.
